# What is Markdown?

To understand what Markdown is, we first need to talk about one of the main problems it solves: **documentation**.

<br />
## What is Documentation?

Documentation is a way of passing knowledge from one source to another.

As humans, we accumulate knowledge through experiences, experiments, and observations. If that knowledge is never documented, it can easily be lost when the person who holds it is no longer around.

That is why documentation is extremely important. It allows knowledge to survive beyond individuals and to be shared across generations.

<br />
## Ways to Document Knowledge

There are many ways you can document something:

- You could record **audio**.
- You could record a **video**.
- You could draw a **diagram or illustration**.
- You could take a **photo** using your phone.
- Or you could use **text**.

Let's focus on **text**.

When using text to document information, there are many ways to structure it. Some text can be **unstructured**, while other formats provide clear structure and organization.

Over the years, smart engineers and researchers have created different structured formats for writing documents. Some common examples include:

- XML
- HTML
- JSON
- Markdown

## So What Is Markdown?

Markdown is a **lightweight markup language** that allows you to write structured documents using plain text.

With Markdown, you can easily create:

- Headings
- Paragraphs
- Lists (ordered and unordered)
- Links
- Images
- Code blocks
- Tables

All of this is done using very simple and readable syntax.

For example, a Markdown file is still easy to read even without being rendered. This makes it extremely convenient for writing documentation.
<br />
## A quick taste of the syntax


```md
# Shopping List

Things I need to shop.

- Bananas
- Eggs
- Milk

You can also add **bold**, *italic*, and `inline code`.
```

<br />
## Why Markdown Became Popular

Markdown has become incredibly popular for several reasons:

- It is **easy to read and write**
- It is **lightweight**
- It can easily be **converted into other formats** such as HTML, PDF, or websites
- It works very well with **version control systems like Git**

Many developers use Markdown to write documentation files such as `README.md` in their repositories.
<br />
## Markdown and AI

Markdown has become even more important in the age of AI.

Many AI agents can easily read and understand Markdown documents. For example, tools like Claude often rely on Markdown files such as `claude.md` to define agent instructions or skills.

In many cases, agent capabilities are simply **best practices documented in Markdown**.

This makes Markdown an incredibly useful skill to learn today.

## Markdown Files

Markdown files are identified by the `.md` file extension.

For example:
- `README.md` - project overview
- `guide.md` - walkthroughs and tutorials
- `documentation.md` - deeper technical docs
